## <a name="value-generation-patterns"></a><span data-ttu-id="2821a-103">Шаблоны для создания значения</span><span class="sxs-lookup"><span data-stu-id="2821a-103">Value Generation Patterns</span></span>

### <a name="no-value-generation"></a><span data-ttu-id="2821a-105">Без создания значение</span><span class="sxs-lookup"><span data-stu-id="2821a-105">No value generation</span></span>

<span data-ttu-id="2821a-106">Без создания значение означает, что всегда будет указано допустимое значение сохраняется в базе данных.</span><span class="sxs-lookup"><span data-stu-id="2821a-106">No value generation means that you will always supply a valid value to be saved to the database.</span></span> <span data-ttu-id="2821a-107">Это допустимое значение должны назначаться новые сущности перед добавлением к контексту.</span><span class="sxs-lookup"><span data-stu-id="2821a-107">This valid value must be assigned to new entities before they are added to the context.</span></span>

### <a name="value-generated-on-add"></a><span data-ttu-id="2821a-108">Добавить значение, созданное в</span><span class="sxs-lookup"><span data-stu-id="2821a-108">Value generated on add</span></span>

<span data-ttu-id="2821a-109">Значение, созданное в надстройка означает генерируется значение для новых сущностей.</span><span class="sxs-lookup"><span data-stu-id="2821a-109">Value generated on add means that a value is generated for new entities.</span></span>

<span data-ttu-id="2821a-110">В зависимости от используемого поставщика базы данных значения могут быть созданные клиентского по EF или в базе данных.</span><span class="sxs-lookup"><span data-stu-id="2821a-110">Depending on the database provider being used, values may be generated client side by EF or in the database.</span></span> <span data-ttu-id="2821a-111">Если это значение формируется базой данных, EF может назначить временное значение при добавлении сущности в контекст.</span><span class="sxs-lookup"><span data-stu-id="2821a-111">If the value is generated by the database, then EF may assign a temporary value when you add the entity to the context.</span></span> <span data-ttu-id="2821a-112">Это временное значение будет выполнена замена по значению создается базой данных во время `SaveChanges()`.</span><span class="sxs-lookup"><span data-stu-id="2821a-112">This temporary value will then be replaced by the database generated value during `SaveChanges()`.</span></span>

<span data-ttu-id="2821a-113">При добавлении сущности в контекст, имеет значение, присваиваемое свойству EF попытается вставить это значение вместо создания нового типа.</span><span class="sxs-lookup"><span data-stu-id="2821a-113">If you add an entity to the context that has a value assigned to the property, then EF will attempt to insert that value rather than generating a new one.</span></span> <span data-ttu-id="2821a-114">Свойство видимости, имеет значение, если он не назначается значение по умолчанию для среды CLR (`null` для `string`, `0` для `int`, `Guid.Empty` для `Guid`и т. д.).</span><span class="sxs-lookup"><span data-stu-id="2821a-114">A property is considered to have a value assigned if it is not assigned the CLR default value (`null` for `string`, `0` for `int`, `Guid.Empty` for `Guid`, etc.).</span></span> <span data-ttu-id="2821a-115">Дополнительные сведения см. в разделе [явные значения создаваемые свойства](../saving/explicit-values-generated-properties.md).</span><span class="sxs-lookup"><span data-stu-id="2821a-115">For more information, see [Explicit values for generated properties](../saving/explicit-values-generated-properties.md).</span></span>

> [!WARNING]  
> <span data-ttu-id="2821a-116">Как это значение формируется для добавленных сущностей будет зависеть от используемого поставщика базы данных.</span><span class="sxs-lookup"><span data-stu-id="2821a-116">How the value is generated for added entities will depend on the database provider being used.</span></span> <span data-ttu-id="2821a-117">Поставщики базы данных автоматически программой установки для создания значения для некоторых типов свойств, но другим пользователям может потребоваться вручную настроить способ создания значение.</span><span class="sxs-lookup"><span data-stu-id="2821a-117">Database providers may automatically setup value generation for some property types, but others may require you to manually setup how the value is generated.</span></span>
>
> <span data-ttu-id="2821a-118">Например, при использовании SQL Server, значения будут создаваться автоматически для `GUID` свойств (с помощью последовательных GUID алгоритм SQL Server).</span><span class="sxs-lookup"><span data-stu-id="2821a-118">For example, when using SQL Server, values will be automatically generated for `GUID` properties (using the SQL Server sequential GUID algorithm).</span></span> <span data-ttu-id="2821a-119">Тем не менее если указать, что `DateTime` свойство создается при добавлении, то необходимо настроить способ создать значения.</span><span class="sxs-lookup"><span data-stu-id="2821a-119">However, if you specify that a `DateTime` property is generated on add, then you must setup a way for the values to be generated.</span></span> <span data-ttu-id="2821a-120">Является одним из способов сделать это, чтобы настроить значение по умолчанию `GETDATE()`, в разделе [значения по умолчанию](relational/default-values.md).</span><span class="sxs-lookup"><span data-stu-id="2821a-120">One way to do this, is to configure a default value of `GETDATE()`, see [Default Values](relational/default-values.md).</span></span>

<span data-ttu-id="2821a-122">Значение, созданное на добавление или обновление означает, что новое значение формируется каждый раз при сохранении записи (insert или update).</span><span class="sxs-lookup"><span data-stu-id="2821a-122">Value generated on add or update means that a new value is generated every time the record is saved (insert or update).</span></span>
